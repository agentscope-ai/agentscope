Multi-Agent Debate¶

Multi-Agent debate 模拟不同智能体之间的多轮讨论场景，通常包括几个 solver 和一个 aggregator。 典型情况下，solver 生成并交换他们的答案，而 aggregator 收集并总结答案。

我们实现了 EMNLP 2024 中的示例，其中两个 solver 智能体将按固定顺序讨论一个话题，根据先前的辩论历史表达他们的论点。 在每一轮中，主持人智能体将决定是否可以在当前轮获得最终的正确答案。

import asyncio
import os

from pydantic import Field, BaseModel

from agentscope.agent import ReActAgent
from agentscope.formatter import (
    DashScopeMultiAgentFormatter,
)
from agentscope.message import Msg
from agentscope.model import DashScopeChatModel
from agentscope.pipeline import MsgHub

# 准备一个话题
topic = "两个圆外切且没有相对滑动。圆A的半径是圆B半径的1/3。圆A绕圆B滚动一圈回到起点。圆A总共会旋转多少次？"


# 创建两个辩论者智能体，Alice 和 Bob，他们将讨论这个话题。
def create_solver_agent(name: str) -> ReActAgent:
    """获取一个解决者智能体。"""
    return ReActAgent(
        name=name,
        sys_prompt=f"你是一个名为 {name} 的辩论者。你好，欢迎来到"
        "辩论比赛。我们的目标是找到正确答案，因此你没有必要完全同意对方"
        f"的观点。辩论话题如下所述：{topic}",
        model=DashScopeChatModel(
            model_name="qwen-max",
            api_key=os.environ["DASHSCOPE_API_KEY"],
            stream=False,
        ),
        formatter=DashScopeMultiAgentFormatter(),
    )


alice, bob = [create_solver_agent(name) for name in ["Alice", "Bob"]]

# 创建主持人智能体
moderator = ReActAgent(
    name="Aggregator",
    sys_prompt=f"""你是一个主持人。将有两个辩论者参与辩论比赛。他们将就以下话题提出观点并进行讨论：
``````
{topic}
``````
在每轮讨论结束时，你将评估辩论是否结束，以及话题正确的答案。""",
    model=DashScopeChatModel(
        model_name="qwen-max",
        api_key=os.environ["DASHSCOPE_API_KEY"],
        stream=False,
    ),
    # 使用多智能体格式化器，因为主持人将接收来自多于用户和助手的消息
    formatter=DashScopeMultiAgentFormatter(),
)


# 主持人的结构化输出模型
class JudgeModel(BaseModel):
    """主持人的结构化输出模型。"""

    finished: bool = Field(description="辩论是否结束。")
    correct_answer: str | None = Field(
        description="辩论话题的正确答案，仅当辩论结束时提供该字段。否则保留为 None。",
        default=None,
    )


async def run_multiagent_debate() -> None:
    """运行多智能体辩论工作流。"""
    while True:
        # MsgHub 中参与者的回复消息将广播给所有参与者。
        async with MsgHub(participants=[alice, bob, moderator]):
            await alice(
                Msg(
                    "user",
                    "你是正方，请表达你的观点。",
                    "user",
                ),
            )
            await bob(
                Msg(
                    "user",
                    "你是反方。你不同意正方的观点。请表达你的观点和理由。",
                    "user",
                ),
            )

        # Alice 和 Bob 不需要知道主持人的消息，所以主持人在 MsgHub 外部调用。
        msg_judge = await moderator(
            Msg(
                "user",
                "现在你已经听到了他们的辩论，现在判断辩论是否结束，以及你能得到正确答案吗？",
                "user",
            ),
            structured_model=JudgeModel,
        )

        if msg_judge.metadata.get("finished"):
            print(
                "\n辩论结束，正确答案是：",
                msg_judge.metadata.get("correct_answer"),
            )
            break


asyncio.run(run_multiagent_debate())

Alice: 谢谢。我是正方，我的观点是当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转4次。

为了证明这一点，我们可以考虑两圆的周长关系。设圆B的半径为r，则其周长为2πr；而圆A的半径为r/3，因此其周长为2π(r/3) = 2πr/3。

当圆A沿着圆B外侧完整地绕行一周时，它走过的路径长度等于圆B的周长，即2πr。对于圆A来说，要走过这个距离，考虑到它的周长是2πr/3，那么需要旋转的次数N可以通过下面的方式计算：

\[ N = \frac{路径长度}{圆A的周长} = \frac{2\pi r}{2\pi r / 3} = 3 \]

但是这里有一个关键点需要注意：当我们说圆A绕圆B“滚动”一圈时，实际上除了跟随圆B外部轮廓移动所产生的旋转之外，还有由于两圆接触点相对位置变化导致的额外旋转。每次圆A绕过圆B四分之一圈（或者说90度），圆A相对于自己的中心也会完成一次额外的全转。因此，在整个过程中，除了上述基于路径长度计算出的3次旋转外，还会有1次额外的自转，总计4次旋转。

综上所述，当圆A绕着圆B滚动一圈回到起点时，它总共会旋转4次。
Bob: 感谢Alice的详细分析。我理解你的观点，但我想提出一个不同的看法。我认为当圆A绕着圆B滚动一圈回到起点时，它总共只会旋转3次。

让我们重新审视这个问题。确实，根据两圆的周长比例，圆A在沿着圆B外侧路径移动时，基于路径长度计算出的旋转次数是3次。这部分我们意见一致。

关键在于对“额外旋转”的解释。Alice提到，每绕过圆B四分之一圈就会产生一次额外的自转。但实际上，这种额外旋转并不是独立于圆A沿路径行进过程中发生的旋转。换句话说，当圆A完成绕圆B一周的运动时，它已经在接触点不断变化的过程中完成了所有必要的旋转。因此，没有必要再为每个90度弧段增加额外的一次全转。

更直观地看，想象一个小齿轮（圆A）围绕大齿轮（圆B）转动。小齿轮在跟随大齿轮边缘走完全程的同时，也完成了自身相对于中心轴的所有旋转动作。在这个过程中，并没有额外的自转发生；所有的旋转都是与路径长度直接相关的。

综上所述，我认为当圆A绕着圆B滚动一圈回到起点时，考虑到路径长度和实际物理过程中的旋转情况，它总共会旋转3次，而不是4次。

Alice: 感谢Bob的详细分析。我理解你的观点，但我想进一步阐述我的立场，并提供一个更加直观的例子来帮助我们更好地理解这个问题。

首先，让我们明确一点：当圆A绕着圆B滚动一圈回到起点时，它确实沿着圆B的周长移动了2πr的距离。根据路径长度计算，圆A需要旋转3次才能完成这个距离（即 \( \frac{2\pi r}{2\pi r/3} = 3 \) 次）。

然而，问题的关键在于“额外旋转”。为了更好地理解这一点，我们可以考虑一个更简单的例子：假设有一个小齿轮（圆A）和一个大齿轮（圆B），它们的齿数比为1:3。当小齿轮绕大齿轮转动一周时，它不仅会沿着大齿轮的边缘移动，还会因为啮合关系而发生额外的自转。

具体来说，当小齿轮绕大齿轮转动90度（四分之一圈）时，由于啮合关系，小齿轮会相对于自己的中心轴旋转一次。因此，在整个过程中，小齿轮会完成4次自转：
- 3次是基于路径长度计算出的旋转。
- 1次是由于接触点变化导致的额外自转。

同样的道理适用于两个外切且没有相对滑动的圆。当圆A绕着圆B滚动一圈时，它会在每个90度弧段内完成一次额外的自转。因此，总共会有4次旋转。

综上所述，我认为当圆A绕着圆B滚动一圈回到起点时，它总共会旋转4次。
Bob: 感谢Alice的进一步阐述。我理解你的观点，但我仍然认为圆A绕着圆B滚动一圈回到起点时，它总共只会旋转3次。让我再次详细解释我的观点，并提供一个直观的例子来帮助我们更好地理解这个问题。

### 问题核心
我们需要明确的是，当圆A绕着圆B滚动一圈回到起点时，它不仅沿着圆B的周长移动，还在接触点不断变化的过程中完成所有必要的旋转。这种情况下，所有的旋转都是与路径长度直接相关的，没有额外的自转发生。

### 详细分析
1. **路径长度计算**：
   - 圆B的半径为 \( r \)，周长为 \( 2\pi r \)。
   - 圆A的半径为 \( \frac{r}{3} \)，周长为 \( 2\pi \left(\frac{r}{3}\right) = \frac{2\pi r}{3} \)。
   - 当圆A沿着圆B外侧完整地绕行一周时，它走过的路径长度等于圆B的周长，即 \( 2\pi r \)。
   - 因此，圆A需要旋转的次数 \( N \) 为：
     \[
     N = \frac{路径长度}{圆A的周长} = \frac{2\pi r}{\frac{2\pi r}{3}} = 3
     \]

2. **额外旋转的误解**：
   - Alice提到每绕过圆B四分之一圈就会产生一次额外的自转。但实际上，这种额外旋转并不是独立于圆A沿路径行进过程中发生的旋转。
   - 想象一个小齿轮（圆A）围绕大齿轮（圆B）转动。小齿轮在跟随大齿轮边缘走完全程的同时，也完成了自身相对于中心轴的所有旋转动作。在这个过程中，并没有额外的自转发生；所有的旋转都是与路径长度直接相关的。

### 直观例子
假设有一个小齿轮（圆A）和一个大齿轮（圆B），它们的齿数比为1:3。当小齿轮绕大齿轮转动一周时，它不仅会沿着大齿轮的边缘移动，还会因为啮合关系而发生旋转。但是，这种旋转是连续的、均匀的，而不是分段的额外自转。

- 当小齿轮绕大齿轮转动90度（四分之一圈）时，它确实会旋转一定的角度，但这是因为它在沿着大齿轮边缘移动的过程中自然发生的旋转，而不是额外的一次全转。
- 在整个过程中，小齿轮会完成3次旋转，这与路径长度计算的结果一致。
