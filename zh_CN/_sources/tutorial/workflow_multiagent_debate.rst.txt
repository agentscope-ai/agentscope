Multi-Agent Debate
========================

Multi-Agent debate 模拟不同智能体之间的多轮讨论场景，通常包括几个 solver 和一个 aggregator。
典型情况下，solver 生成并交换他们的答案，而 aggregator 收集并总结答案。

我们实现了 `EMNLP 2024`_ 中的示例，其中两个 solver 智能体将按固定顺序讨论一个话题，根据先前的辩论历史表达他们的论点。
在每一轮中，主持人智能体将决定是否可以在当前轮获得最终的正确答案。

    import asyncio
    import os

    from pydantic import Field, BaseModel

    from agentscope.agent import ReActAgent
    from agentscope.formatter import (
        DashScopeMultiAgentFormatter,
    )
    from agentscope.message import Msg
    from agentscope.model import DashScopeChatModel
    from agentscope.pipeline import MsgHub

    # 准备一个话题
    topic = "两个圆外切且没有相对滑动。圆A的半径是圆B半径的1/3。圆A绕圆B滚动一圈回到起点。圆A总共会旋转多少次？"


    # 创建两个辩论者智能体，Alice 和 Bob，他们将讨论这个话题。
    def create_solver_agent(name: str) -> ReActAgent:
        """获取一个解决者智能体。"""
        return ReActAgent(
            name=name,
            sys_prompt=f"你是一个名为 {name} 的辩论者。你好，欢迎来到"
            "辩论比赛。我们的目标是找到正确答案，因此你没有必要完全同意对方"
            f"的观点。辩论话题如下所述：{topic}",
            model=DashScopeChatModel(
                model_name="qwen-max",
                api_key=os.environ["DASHSCOPE_API_KEY"],
                stream=False,
            ),
            formatter=DashScopeMultiAgentFormatter(),
        )


    alice, bob = [create_solver_agent(name) for name in ["Alice", "Bob"]]

    # 创建主持人智能体
    moderator = ReActAgent(
        name="Aggregator",
        sys_prompt=f"""你是一个主持人。将有两个辩论者参与辩论比赛。他们将就以下话题提出观点并进行讨论：
    ``````
    {topic}
    ``````
    在每轮讨论结束时，你将评估辩论是否结束，以及话题正确的答案。""",
        model=DashScopeChatModel(
            model_name="qwen-max",
            api_key=os.environ["DASHSCOPE_API_KEY"],
            stream=False,
        ),
        # 使用多智能体格式化器，因为主持人将接收来自多于用户和助手的消息
        formatter=DashScopeMultiAgentFormatter(),
    )


    # 主持人的结构化输出模型
    class JudgeModel(BaseModel):
        """主持人的结构化输出模型。"""

        finished: bool = Field(description="辩论是否结束。")
        correct_answer: str | None = Field(
            description="辩论话题的正确答案，仅当辩论结束时提供该字段。否则保留为 None。",
            default=None,
        )


    async def run_multiagent_debate() -> None:
        """运行多智能体辩论工作流。"""
        while True:
            # MsgHub 中参与者的回复消息将广播给所有参与者。
            async with MsgHub(participants=[alice, bob, moderator]):
                await alice(
                    Msg(
                        "user",
                        "你是正方，请表达你的观点。",
                        "user",
                    ),
                )
                await bob(
                    Msg(
                        "user",
                        "你是反方。你不同意正方的观点。请表达你的观点和理由。",
                        "user",
                    ),
                )

            # Alice 和 Bob 不需要知道主持人的消息，所以主持人在 MsgHub 外部调用。
            msg_judge = await moderator(
                Msg(
                    "user",
                    "现在你已经听到了他们的辩论，现在判断辩论是否结束，以及你能得到正确答案吗？",
                    "user",
                ),
                structured_model=JudgeModel,
            )

            if msg_judge.metadata.get("finished"):
                print(
                    "\n辩论结束，正确答案是：",
                    msg_judge.metadata.get("correct_answer"),
                )
                break


    asyncio.run(run_multiagent_debate())

    Alice: 正方观点是：当圆A绕着圆B外切滚动一圈回到起点时，圆A总共会旋转4次。

    解释如下：设圆B的半径为R，则圆A的半径为r = R/3。当圆A绕圆B滚动一周时，圆心A走过的路径长度等于圆B的周长，即2πR。由于圆A与圆B之间没有相对滑动，所以圆A边缘上的某一点在绕圆B一周的过程中所走过的总距离也是2πR。

    然而，对于圆A本身来说，它自身也必须旋转一定的次数来完成这个过程。我们知道，圆A自转一周的距离是其自身的周长，也就是2πr = 2π(R/3)。因此，要计算圆A自转了多少圈，我们可以将圆心A走过的总路程除以圆A的周长：

    \[ \text{圆A自转的圈数} = \frac{\text{圆A边缘点走过的总距离}}{\text{圆A的周长}} = \frac{2\pi R}{2\pi (R/3)} = \frac{R}{R/3} = 3 \]

    但是，这里我们只考虑了圆A围绕圆B中心旋转的影响。实际上，当圆A绕圆B滚动一圈的同时，圆A还会因为自身与圆B之间的相对位置变化而额外旋转一次（想象一下，如果把圆A放在桌子上滚，当它滚回原位时，它也会自转了一圈）。因此，除了跟随圆B中心的3次旋转外，还应该加上这1次额外的自转，总共是4次旋转。

    综上所述，当圆A绕圆B滚动一圈回到起点时，圆A总共会旋转4次。
    Bob: 反方观点是：当圆A绕着圆B外切滚动一圈回到起点时，圆A总共会旋转3次，而不是4次。

    解释如下：设圆B的半径为R，那么圆A的半径为r = R/3。当圆A沿着圆B的外部边缘无滑动地滚动一周时，圆心A走过的路径长度确实是圆B的周长，即2πR。由于没有相对滑动，圆A边缘上的某一点在绕圆B一周的过程中所经过的总距离也是2πR。

    对于圆A来说，它自转的距离是其自身的周长，即2πr或2π(R/3)。为了计算圆A自转了多少圈，我们需要将圆心A走过的总路程除以圆A的周长：

    \[ \text{圆A自转的圈数} = \frac{\text{圆A边缘点走过的总距离}}{\text{圆A的周长}} = \frac{2\pi R}{2\pi (R/3)} = 3 \]

    这里的关键在于理解这个过程中的几何关系。圆A绕着圆B滚动一周，并且圆A与圆B之间没有滑动，这意味着圆A在绕圆B中心旋转的同时也在进行自转。但是，当我们考虑圆A绕圆B中心旋转时，实际上这已经包含了圆A所有必要的旋转。这是因为圆A不仅需要跟随圆B的曲线运动，而且还要保证接触点始终不变，这就意味着圆A必须完成整数倍于自身周长的旋转来保持这种接触。

    因此，在整个过程中，圆A不需要额外的一次完整自转。圆A跟随圆B的曲线所做的3次旋转已经足够使它回到初始位置并且恢复到开始时的状态。所以，圆A总共会旋转3次。这与正方提到的“额外一次自转”不同，因为那部分已经被包含在了跟随圆B中心旋转的次数中。
    Aggregator: 在听取了双方的辩论后，我们可以确定的是，当圆A绕着圆B外切滚动一圈回到起点时，圆A总共会旋转3次。Alice提到的额外一次自转实际上已经在跟随圆B中心旋转的过程中被考虑进去了。Bob的观点正确地指出了这一点：圆A为了保持与圆B接触点的一致性而进行的旋转已经被计算在内了，因此不需要再加上一个额外的完整自转。所以，根据几何学和运动学原理，答案是3次旋转。

    辩论结束，正确答案是： 3次
