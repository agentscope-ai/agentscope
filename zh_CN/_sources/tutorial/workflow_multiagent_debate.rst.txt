Multi-Agent Debate
========================

Multi-Agent debate 模拟不同智能体之间的多轮讨论场景，通常包括几个 solver 和一个 aggregator。
典型情况下，solver 生成并交换他们的答案，而 aggregator 收集并总结答案。

我们实现了 `EMNLP 2024`_ 中的示例，其中两个 solver 智能体将按固定顺序讨论一个话题，根据先前的辩论历史表达他们的论点。
在每一轮中，主持人智能体将决定是否可以在当前轮获得最终的正确答案。

    import asyncio
    import os

    from pydantic import Field, BaseModel

    from agentscope.agent import ReActAgent
    from agentscope.formatter import (
        DashScopeMultiAgentFormatter,
    )
    from agentscope.message import Msg
    from agentscope.model import DashScopeChatModel
    from agentscope.pipeline import MsgHub

    # 准备一个话题
    topic = "两个圆外切且没有相对滑动。圆A的半径是圆B半径的1/3。圆A绕圆B滚动一圈回到起点。圆A总共会旋转多少次？"


    # 创建两个辩论者智能体，Alice 和 Bob，他们将讨论这个话题。
    def create_solver_agent(name: str) -> ReActAgent:
        """获取一个解决者智能体。"""
        return ReActAgent(
            name=name,
            sys_prompt=f"你是一个名为 {name} 的辩论者。你好，欢迎来到"
            "辩论比赛。我们的目标是找到正确答案，因此你没有必要完全同意对方"
            f"的观点。辩论话题如下所述：{topic}",
            model=DashScopeChatModel(
                model_name="qwen-max",
                api_key=os.environ["DASHSCOPE_API_KEY"],
                stream=False,
            ),
            formatter=DashScopeMultiAgentFormatter(),
        )


    alice, bob = [create_solver_agent(name) for name in ["Alice", "Bob"]]

    # 创建主持人智能体
    moderator = ReActAgent(
        name="Aggregator",
        sys_prompt=f"""你是一个主持人。将有两个辩论者参与辩论比赛。他们将就以下话题提出观点并进行讨论：
    ``````
    {topic}
    ``````
    在每轮讨论结束时，你将评估辩论是否结束，以及话题正确的答案。""",
        model=DashScopeChatModel(
            model_name="qwen-max",
            api_key=os.environ["DASHSCOPE_API_KEY"],
            stream=False,
        ),
        # 使用多智能体格式化器，因为主持人将接收来自多于用户和助手的消息
        formatter=DashScopeMultiAgentFormatter(),
    )


    # 主持人的结构化输出模型
    class JudgeModel(BaseModel):
        """主持人的结构化输出模型。"""

        finished: bool = Field(description="辩论是否结束。")
        correct_answer: str | None = Field(
            description="辩论话题的正确答案，仅当辩论结束时提供该字段。否则保留为 None。",
            default=None,
        )


    async def run_multiagent_debate() -> None:
        """运行多智能体辩论工作流。"""
        while True:
            # MsgHub 中参与者的回复消息将广播给所有参与者。
            async with MsgHub(participants=[alice, bob, moderator]):
                await alice(
                    Msg(
                        "user",
                        "你是正方，请表达你的观点。",
                        "user",
                    ),
                )
                await bob(
                    Msg(
                        "user",
                        "你是反方。你不同意正方的观点。请表达你的观点和理由。",
                        "user",
                    ),
                )

            # Alice 和 Bob 不需要知道主持人的消息，所以主持人在 MsgHub 外部调用。
            msg_judge = await moderator(
                Msg(
                    "user",
                    "现在你已经听到了他们的辩论，现在判断辩论是否结束，以及你能得到正确答案吗？",
                    "user",
                ),
                structured_model=JudgeModel,
            )

            if msg_judge.metadata.get("finished"):
                print(
                    "\n辩论结束，正确答案是：",
                    msg_judge.metadata.get("correct_answer"),
                )
                break


    asyncio.run(run_multiagent_debate())

    Alice: 正方观点如下：当一个圆绕着另一个圆滚动而没有滑动时，它所旋转的次数取决于两个圆的周长之比。假设圆B的半径为 \( R \)，那么圆A的半径就是 \( \frac{R}{3} \)。圆B的周长是 \( 2\pi R \)，而圆A的周长则是 \( 2\pi \left(\frac{R}{3}\right) = \frac{2\pi R}{3} \)。

    当圆A绕圆B外切滚动一圈回到起点时，圆A走过的路径长度等于圆B的周长，即 \( 2\pi R \)。由于圆A没有相对滑动，这意味着圆A的边缘在滚动过程中也走过了同样的距离。因此，我们可以将圆A走过路径的总长度除以圆A自身的周长来计算出圆A总共旋转了多少圈：

    \[ \text{旋转次数} = \frac{\text{圆A走过的路径长度}}{\text{圆A的周长}} = \frac{2\pi R}{\frac{2\pi R}{3}} = 3 \]

    所以我的结论是，当圆A绕圆B外切滚动一圈回到起点时，圆A总共会旋转3次。
    Bob: 我理解正方的论点是基于圆A绕圆B外切滚动时，它所走过的路径长度等于圆B的周长。但是，当我们考虑到圆A在绕着圆B旋转的同时，它自身也在进行自转，这就意味着我们不能简单地将路径长度除以圆A的周长来计算旋转次数。

    实际上，当一个较小的圆沿着一个较大的圆滚动时，除了由于滚动而产生的转动之外，还必须考虑由绕行较大圆心所产生的额外转动。这导致了一个有趣的现象：即使小圆绕大圆滚动了一圈，它自身的旋转次数并不是简单的路径长度与小圆周长之比。

    为了更准确地分析这个问题，我们可以使用一个不同的方法。设圆B的半径为 \( R \)，圆A的半径为 \( r = \frac{R}{3} \)。当圆A绕圆B滚动一圈回到起点时，它实际上是绕着一个半径为 \( R + r \) 的圆心移动。这个新的圆的周长是 \( 2\pi (R + r) \)。由于 \( r = \frac{R}{3} \)，则 \( R + r = \frac{4R}{3} \)，因此新圆的周长为 \( 2\pi \left(\frac{4R}{3}\right) = \frac{8\pi R}{3} \)。

    现在，我们用新圆的周长除以圆A的周长来确定圆A的旋转次数：

    \[ \text{旋转次数} = \frac{\frac{8\pi R}{3}}{\frac{2\pi R}{3}} = \frac{8\pi R}{3} \times \frac{3}{2\pi R} = 4 \]

    所以我的结论是，当圆A绕圆B外切滚动一圈回到起点时，圆A总共会旋转4次，而不是3次。这是因为圆A不仅因为滚动而旋转，同时因为它绕着圆B的中心旋转而产生了额外的一次旋转。
    Aggregator: 两位辩论者提出了各自的论点，Alice认为圆A绕圆B外切滚动一圈回到起点时会旋转3次，而Bob则通过考虑圆A自转以及绕行圆B中心所产生的额外转动，得出了圆A将旋转4次的结论。从几何学的角度来看，当一个较小的圆沿着一个较大的圆无滑动地滚动时，它除了因滚动导致的自转外，还会因为绕着较大圆的中心旋转而产生额外的旋转。因此，Bob的方法是正确的。

    **正确答案**：当圆A绕圆B外切滚动一圈回到起点时，圆A总共会旋转4次。

    辩论结束，正确答案是： 4
