Multi-Agent Debate
========================

Multi-Agent debate 模拟不同智能体之间的多轮讨论场景，通常包括几个 solver 和一个 aggregator。
典型情况下，solver 生成并交换他们的答案，而 aggregator 收集并总结答案。

我们实现了 `EMNLP 2024`_ 中的示例，其中两个 solver 智能体将按固定顺序讨论一个话题，根据先前的辩论历史表达他们的论点。
在每一轮中，主持人智能体将决定是否可以在当前轮获得最终的正确答案。

    import asyncio
    import os

    from pydantic import Field, BaseModel

    from agentscope.agent import ReActAgent
    from agentscope.formatter import (
        DashScopeMultiAgentFormatter,
    )
    from agentscope.message import Msg
    from agentscope.model import DashScopeChatModel
    from agentscope.pipeline import MsgHub

    # 准备一个话题
    topic = "两个圆外切且没有相对滑动。圆A的半径是圆B半径的1/3。圆A绕圆B滚动一圈回到起点。圆A总共会旋转多少次？"


    # 创建两个辩论者智能体，Alice 和 Bob，他们将讨论这个话题。
    def create_solver_agent(name: str) -> ReActAgent:
        """获取一个解决者智能体。"""
        return ReActAgent(
            name=name,
            sys_prompt=f"你是一个名为 {name} 的辩论者。你好，欢迎来到"
            "辩论比赛。我们的目标是找到正确答案，因此你没有必要完全同意对方"
            f"的观点。辩论话题如下所述：{topic}",
            model=DashScopeChatModel(
                model_name="qwen-max",
                api_key=os.environ["DASHSCOPE_API_KEY"],
                stream=False,
            ),
            formatter=DashScopeMultiAgentFormatter(),
        )


    alice, bob = [create_solver_agent(name) for name in ["Alice", "Bob"]]

    # 创建主持人智能体
    moderator = ReActAgent(
        name="Aggregator",
        sys_prompt=f"""你是一个主持人。将有两个辩论者参与辩论比赛。他们将就以下话题提出观点并进行讨论：
    ``````
    {topic}
    ``````
    在每轮讨论结束时，你将评估辩论是否结束，以及话题正确的答案。""",
        model=DashScopeChatModel(
            model_name="qwen-max",
            api_key=os.environ["DASHSCOPE_API_KEY"],
            stream=False,
        ),
        # 使用多智能体格式化器，因为主持人将接收来自多于用户和助手的消息
        formatter=DashScopeMultiAgentFormatter(),
    )


    # 主持人的结构化输出模型
    class JudgeModel(BaseModel):
        """主持人的结构化输出模型。"""

        finished: bool = Field(description="辩论是否结束。")
        correct_answer: str | None = Field(
            description="辩论话题的正确答案，仅当辩论结束时提供该字段。否则保留为 None。",
            default=None,
        )


    async def run_multiagent_debate() -> None:
        """运行多智能体辩论工作流。"""
        while True:
            # MsgHub 中参与者的回复消息将广播给所有参与者。
            async with MsgHub(participants=[alice, bob, moderator]):
                await alice(
                    Msg(
                        "user",
                        "你是正方，请表达你的观点。",
                        "user",
                    ),
                )
                await bob(
                    Msg(
                        "user",
                        "你是反方。你不同意正方的观点。请表达你的观点和理由。",
                        "user",
                    ),
                )

            # Alice 和 Bob 不需要知道主持人的消息，所以主持人在 MsgHub 外部调用。
            msg_judge = await moderator(
                Msg(
                    "user",
                    "现在你已经听到了他们的辩论，现在判断辩论是否结束，以及你能得到正确答案吗？",
                    "user",
                ),
                structured_model=JudgeModel,
            )

            if msg_judge.metadata.get("finished"):
                print(
                    "\n辩论结束，正确答案是：",
                    msg_judge.metadata.get("correct_answer"),
                )
                break


    asyncio.run(run_multiagent_debate())

    Alice: 正方观点是：当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转4次。

    解释如下：设圆B的半径为R，则圆A的半径为r=R/3。当圆A绕圆B外切滚动而没有滑动时，圆A的中心将沿着一个半径为R+r的圆周运动。这意味着圆A的中心走过的路径长度等于(R+r)乘以2π，即(4/3)R*2π。

    对于圆A自身来说，它每自转一圈，其边缘点就会走过一个距离等于它的周长，也就是2πr = 2π(R/3)。因此，要计算圆A在绕圆B滚动一圈的过程中自转了多少圈，我们需要用圆A中心走过的总距离除以圆A的周长：

    \[ \text{圆A自转圈数} = \frac{(4/3)R \times 2\pi}{(R/3) \times 2\pi} = \frac{4/3 R}{1/3 R} = 4 \]

    所以，根据这个分析，圆A在绕圆B滚动一圈回到起点时会自转4次。
    Bob: 反方观点是：当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转3次。

    解释如下：设圆B的半径为R，则圆A的半径为r=R/3。当圆A绕着圆B外切滚动而没有滑动时，圆A的中心沿着一个半径为R+r的轨迹运动，即(R+R/3)=4R/3的圆形路径。因此，圆A的中心所走过的距离是(4R/3) * 2π。

    然而，关键在于理解圆A在绕圆B旋转的同时也在自转。当我们说圆A绕圆B“滚动一圈”，实际上是指圆A的接触点与圆B的接触点重新对齐。由于圆A和圆B的半径比为1:3，圆A必须绕自己的轴心旋转3圈才能使它的接触点与圆B上的接触点再次对齐（想象一下齿轮，小齿轮要旋转多圈才能让大齿轮转动一圈）。

    因此，圆A在绕圆B一周的过程中，它不仅需要围绕圆B的中心旋转，而且还需要自转以保持与圆B的接触。所以，圆A的总旋转次数是：

    \[ \text{圆A自转圈数} = \frac{\text{圆A中心走过的距离}}{\text{圆A的周长}} - 1 \]

    \[ \text{圆A自转圈数} = \frac{(4/3)R \times 2\pi}{(R/3) \times 2\pi} - 1 = 4 - 1 = 3 \]

    这里减去的1代表的是圆A围绕圆B的公转，剩下的3就是圆A自身的自转。因此，根据这个分析，圆A在绕圆B滚动一圈回到起点时会自转3次。这与正方观点不同，因为我们考虑了圆A在绕圆B公转时也发生了自转。
    Bob: 反方观点认为，当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转3次。这是因为圆A在绕圆B一周过程中，除了绕圆B的公转之外，还必须自转来保持与圆B的接触。由于圆A的半径是圆B的1/3，为了使它们的接触点重新对齐，圆A需要自转3圈。因此，圆A的总旋转次数是其绕圆B公转的一圈加上自转的3圈，再减去公转的那一次，结果是3次自转。
    Aggregator: 两位辩论者提出了各自的观点，但是让我们更仔细地分析一下这个问题。当圆A绕着圆B滚动时，它的运动包括了两个部分：公转和自转。Alice提到的圆A中心走过的路径长度是正确的，也就是(4/3)R*2π。但关键在于理解圆A完成一次绕行时，它同时进行的自转次数。

    Bob指出，圆A在绕圆B一周过程中，除了绕圆B的公转之外，还必须自转来保持与圆B的接触。这个观点也是正确的。然而，计算自转圈数时，我们不需要从总旋转中减去公转的那一圈。因为圆A每绕圆B一圈，实际上就相当于它沿着一个更大的圆周长（即(R+r) * 2π）滚动了一次，而在这个过程中，它需要自转的圈数正好是它半径相对于圆B的比例，即3倍于自身的周长。

    因此，正确答案是这样的：圆A绕圆B滚动一圈回到起点时，由于圆A的半径是圆B的1/3，所以圆A会自转3次。这是因为圆A绕着圆B滚动的同时，它自己的边缘点也需要走过一个等于圆B周长的距离，这导致了额外的自转。综上所述，圆A总共会旋转3次。

    辩论结束，正确答案是： 3次
