Multi-Agent Debate
========================

Multi-Agent debate 模拟不同智能体之间的多轮讨论场景，通常包括几个 solver 和一个 aggregator。
典型情况下，solver 生成并交换他们的答案，而 aggregator 收集并总结答案。

我们实现了 `EMNLP 2024`_ 中的示例，其中两个 solver 智能体将按固定顺序讨论一个话题，根据先前的辩论历史表达他们的论点。
在每一轮中，主持人智能体将决定是否可以在当前轮获得最终的正确答案。

    import asyncio
    import os

    from pydantic import Field, BaseModel

    from agentscope.agent import ReActAgent
    from agentscope.formatter import (
        DashScopeMultiAgentFormatter,
    )
    from agentscope.message import Msg
    from agentscope.model import DashScopeChatModel
    from agentscope.pipeline import MsgHub

    # 准备一个话题
    topic = "两个圆外切且没有相对滑动。圆A的半径是圆B半径的1/3。圆A绕圆B滚动一圈回到起点。圆A总共会旋转多少次？"


    # 创建两个辩论者智能体，Alice 和 Bob，他们将讨论这个话题。
    def create_solver_agent(name: str) -> ReActAgent:
        """获取一个解决者智能体。"""
        return ReActAgent(
            name=name,
            sys_prompt=f"你是一个名为 {name} 的辩论者。你好，欢迎来到"
            "辩论比赛。我们的目标是找到正确答案，因此你没有必要完全同意对方"
            f"的观点。辩论话题如下所述：{topic}",
            model=DashScopeChatModel(
                model_name="qwen-max",
                api_key=os.environ["DASHSCOPE_API_KEY"],
                stream=False,
            ),
            formatter=DashScopeMultiAgentFormatter(),
        )


    alice, bob = [create_solver_agent(name) for name in ["Alice", "Bob"]]

    # 创建主持人智能体
    moderator = ReActAgent(
        name="Aggregator",
        sys_prompt=f"""你是一个主持人。将有两个辩论者参与辩论比赛。他们将就以下话题提出观点并进行讨论：
    ``````
    {topic}
    ``````
    在每轮讨论结束时，你将评估辩论是否结束，以及话题正确的答案。""",
        model=DashScopeChatModel(
            model_name="qwen-max",
            api_key=os.environ["DASHSCOPE_API_KEY"],
            stream=False,
        ),
        # 使用多智能体格式化器，因为主持人将接收来自多于用户和助手的消息
        formatter=DashScopeMultiAgentFormatter(),
    )


    # 主持人的结构化输出模型
    class JudgeModel(BaseModel):
        """主持人的结构化输出模型。"""

        finished: bool = Field(description="辩论是否结束。")
        correct_answer: str | None = Field(
            description="辩论话题的正确答案，仅当辩论结束时提供该字段。否则保留为 None。",
            default=None,
        )


    async def run_multiagent_debate() -> None:
        """运行多智能体辩论工作流。"""
        while True:
            # MsgHub 中参与者的回复消息将广播给所有参与者。
            async with MsgHub(participants=[alice, bob, moderator]):
                await alice(
                    Msg(
                        "user",
                        "你是正方，请表达你的观点。",
                        "user",
                    ),
                )
                await bob(
                    Msg(
                        "user",
                        "你是反方。你不同意正方的观点。请表达你的观点和理由。",
                        "user",
                    ),
                )

            # Alice 和 Bob 不需要知道主持人的消息，所以主持人在 MsgHub 外部调用。
            msg_judge = await moderator(
                Msg(
                    "user",
                    "现在你已经听到了他们的辩论，现在判断辩论是否结束，以及你能得到正确答案吗？",
                    "user",
                ),
                structured_model=JudgeModel,
            )

            if msg_judge.metadata.get("finished"):
                print(
                    "\n辩论结束，正确答案是：",
                    msg_judge.metadata.get("correct_answer"),
                )
                break


    asyncio.run(run_multiagent_debate())

    Alice: 正方的观点是，当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转4次。为了证明这一点，我们可以使用圆的周长和相对运动的概念来解释。

    设圆B的半径为 \( R \)，则圆A的半径为 \( \frac{R}{3} \)。圆B的周长为 \( 2\pi R \)，而圆A的周长则是 \( 2\pi \times \frac{R}{3} = \frac{2\pi R}{3} \)。

    当圆A绕着圆B无滑动地滚动一周时，圆A的质心（即圆心）沿着圆B的外围移动了距离 \( 2\pi R \)。由于圆A在滚动过程中没有滑动，它所走过的路径长度等于其自身周长乘以它自转的次数。因此，如果圆A自转了 \( n \) 次，则有：

    \[ n \times \frac{2\pi R}{3} = 2\pi R \]

    解这个等式得到：

    \[ n = \frac{2\pi R}{(2\pi R / 3)} = 3 \]

    但是，这里有一个额外的因素需要考虑：当圆A绕圆B外切滚动时，除了围绕圆B中心旋转的这3次之外，圆A还会因为跟随圆B的圆形轨迹而进行一次额外的转动。这是因为圆A在绕行圆B的同时，也在随着圆B的曲率进行一个完整的旋转。所以，实际上圆A总共会旋转3次加上这次额外的旋转，总计4次。
    Bob: 反方的观点是，当圆A绕着圆B滚动一圈回到起点时，圆A总共会旋转3次，而不是4次。让我解释一下为什么。

    首先，我们同意正方关于圆A和圆B周长的计算。圆B的周长为 \(2\pi R\)，而圆A的周长为 \(\frac{2\pi R}{3}\)。当圆A绕圆B无滑动地滚动一周时，它的质心确实沿着圆B的外围移动了距离 \(2\pi R\)。由于没有相对滑动，我们可以得出圆A必须自转的次数 \(n\) 以覆盖这段距离：

    \[ n \times \frac{2\pi R}{3} = 2\pi R \]

    解这个等式得到：

    \[ n = \frac{2\pi R}{(2\pi R / 3)} = 3 \]

    这说明圆A在绕圆B外切滚动一周的过程中，它自身会旋转3次以完成整个路径。这里的关键在于理解额外的一次转动实际上是已经包含在这3次中的。

    当一个较小的圆绕着一个较大的圆滚动时，较小的圆不仅因为其自身的旋转而前进，还因为它跟随较大圆的曲线而经历了一个几何上的旋转。然而，这种“跟随”式的旋转实际上就是圆A相对于地面的旋转，而不是一个独立的、额外的旋转。因此，在圆A绕圆B一周的过程中，它与地面接触点的轨迹表明它只进行了3次旋转。

    所以，根据以上分析，我方认为圆A绕圆B滚动一圈回到起点时，圆A总共会旋转3次。没有必要加上所谓的“额外”的一次旋转，因为那其实已经在3次旋转中被考虑进去了。
    Bob: 经过进一步的反思，我认为有必要澄清一点：当圆A绕着圆B滚动的时候，它不仅会因为要走过圆B的周长而旋转3次，还会因为在绕行过程中跟随圆B的曲率进行额外的一次完整旋转。这种额外的旋转并不是基于圆A的半径，而是基于圆A绕着圆B中心做圆周运动的事实。

    想象一下，如果我们将圆A看作是一个小球，并且它固定在一个大轮子（圆B）上，那么当我们转动大轮子时，小球除了绕着自己的轴旋转之外，还会随着大轮子的转动而旋转。这种现象在物理上被称为刚体的复合运动，其中包含了自转和平移两种成分。

    因此，虽然圆A为了覆盖圆B的周长需要旋转3次，但同时它也会因为跟随圆B的圆形轨迹而进行第4次旋转。这就是为什么我认为最终圆A会旋转4次的原因。
    Aggregator: 两位辩论者已经清楚地解释了圆A绕圆B滚动一圈回到起点时，它总共会旋转的次数。他们指出了圆A不仅因为要走过圆B的周长而旋转3次，而且由于跟随圆B的圆形轨迹而进行额外的一次完整旋转。因此，圆A总共会旋转4次。

    辩论结束，正确答案是： 4次
