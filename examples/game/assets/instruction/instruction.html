游戏设计主要包含角色设计和剧情设计两部分。

角色设计

新角色创建流程如下：

  1. 点击“🆕创建角色”
  2. 角色配置
  3. 点击“🛄保存角色”

一下是如何在角色配置页面配置角色的介绍。

角色基础配置

  1. 角色名称：必须输入，如果有同名，将会覆盖原配置。
  2. 角色头像：支持本地上传和llm生成。
  3. 熟悉程度：【陌生，稍微熟悉，熟悉】该值影响角色对玩家透露的信息程度
  4. 是否开启记忆功能：默认开启

  5. 模型设置：即agent调用的模型，支持post_api和tongyi

    角色特征设置

  6. 食物偏好：当该角色出现在“营业环节”，会综合此信息和当天食材生成要求菜品；当玩家搭配出类似的菜品，可以提高该角色熟悉度以使角色透露更多的信息。

  7. 背景介绍：提供基本的人物认定，会影响人物在游戏中的回答。

    角色剧情设置

    线索卡是游戏剧情的核心。

  8. 【线索卡=剧情】每张线索卡含有该角色在某段剧情中（剧情ID）的视角、想法。

  9. 【主角线索卡插入】如果该角色是主角，那所有线索卡会从一开始就插入。
  10. 【配角线索卡插入】如果该角色是配角，玩家需要在“营业环节”做出让该角色满意的菜品，逐步让线索卡插入角色记忆中。（所以设计者需要考虑线索卡的顺序！）
  11. 【线索卡揭露】当玩家和角色对话被判定为包含改线索，线索卡会在游戏界面展示给玩家。

剧情/关卡设计

每个剧情由以下环节组成：

  1. 开场
  2. 营业环节
  3. 解密环节

其中“开场”是固定环节，由玩家和主角交互； “营业环节”和“解谜环节”是可选环节。

剧情基本设置

  1. 剧情id：用于识别确定剧情，需要是整数数字
  2. 剧情任务：剧情的名字
  3. 尝试次数：最多尝试解答谜题的次数
  4. 配置主角/配置配角：在已经配置好的角色中选择主角/配角。
    1. 主角会出现在“开场”阶段，交代剧情背景和可以提供玩家的一些关键信息。
    2. 配角会出现在“营业环节”和“解密环节”。
  5. 剧情任务:剧情环节可选选择:
    1. 营业环节：配角出现在营业环节，并且会点菜。配角对菜满意的话，主动揭露一张线索卡，并且和玩家继续对话。
    2. 解密环节：
      1. 当没有配角的的时候，玩家会进入与主角对话；
      2. 当有配角的时候，玩家可以邀请主角 + 配角进入多角色对话。

剧情依赖关系

  1. 配置前置剧情：当前置剧情完成后，当前剧情才可能被激活。

  2. 设置解锁剧情：

    1. 默认解锁方式是“always_true” （将来会引入更多判断条件）
    2. 解锁剧情ID：当前剧情成功后，被激活的下一个剧情。

      剧情完成条件/判断

  3. 任务描述：让玩家知道当前剧情的任务主要是什么。同时也会出现在游戏界面的“任务”tab中

  4. 剧情完成条件：让LLM来判定玩家的回答、与角色的对话是否满足该条件；如果是，则该任务会被判定完成。

剧情开场阶段设定

为了更好控制剧情，开发者应该提供以下信息让玩家更快融入

  1. 开场白
  2. 开场白配图
  3. 主角入场发言
  4. 玩家初始发言提示
  5. 主角退场发言
